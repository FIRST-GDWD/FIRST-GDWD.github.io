This project is meant to be the culmination of everything you've learned so far about CSS.  It is one of our first forays into layout building, so it may seem a bit intimidating, but by now, you've learned all the tools and features of CSS needed to build such a layout.

This project is meant to be challenging.  You will most likely write a lot of code that doesn't behave the way you want it to the first time; that is okay.  Trial and error is part of coding (even when you do it professionally!), and it is also how you learn.  Mentally prepare yourself for small failures along the way, so that you can pick yourself up again with a positive attitude and try something else.  The only time you truly fail is when you give up.  

And don't forget, your instructor is here to help you if you get stuck, so don't be afraid to ask for hints or help debugging!

 

If you're unsure about how to unzip the project template below, or how to get it into your repository, check out the following tutorial from the Coding Cookbook - Importing a Code Template.

 

    1. Download the Generic License template and unzip it; then move it into your repository under the name "generic-license".

      • Our grading focus is going to be on the layout of the license elements
        • meaning, we're going to be looking at whether the elements are in the right places and next to the appropriate content
      • As a result, if you want to change the styling of the license to use different colors or font sizes, or you want to change the content to include different text or images, you are free to do so. 
        • In fact, we encourage you to change things up, once you've completed the main layout!

 

TIPS

    • We STRONGLY recommend that you look over the following "appetizer" for the project, to guide you through the process of how to mentally breakdown the layout into smaller bite-size pieces.
      • Identifying Layout Boxes
      • This walkthrough practically gives you a visual guide regarding the elements you need to transform and the order to change them in.
        • You are not required to take this approach, but it will probably be the easiest!
    • As the walkthrough above suggests, we recommend you try to tackle the CSS from the outermost containers to the innermost containers
      • Remember that elements that should take up a whole line in their container should be block elements (e.g. <div>), while elements that go side-by-side will likely need to be changed into inline-block elements (with display: inline-block;).
      • Some elements may also need to be explicitly sized, either with absolute units (e.g. width: 700px;) or relative units (e.g. width: 65%)
        • Just remember that relatively sized units (e.g. width: 34%) are sized relative to their containers; if their immediate containers aren't sized explicitly, they're going to keep going up the code until they find a container that does have an explicit size.
        • Applying a width of 700px to the license itself is a good starting point!
    • Pro-tip: if you struggle with knowing whether you're targeting the correct element in CSS or not, start by adding a border to it.  If that border contains the elements you expect, you probably targeted it correctly!  Just remember to remove the border when you're done styling (unless you want to have a border).
      • In fact, for this project, you may want to start by creating borders that match the images in the "Identifying Layout Boxes" walkthrough above, just to make sure you're targeting the correct elements before trying to get them side-by-side
    • The HTML code utilizes the Font Awesome Icon library.  If you'd like to view a refresher on how to use that, either to add additional icons or just for the sake of refreshing your memory, check out the tutorial below:

GOOGLE FONTS

You may have noticed that the text for the signature in the license screenshot looks different from any of the generic font families we learned about before.  That's because it isn't using a generic font family!

Instead, we're taking advantage of a service offered by Google called Google Fonts, which allows you to use any fonts from their library and practically guarantee that they will work for anyone viewing your web page!  This is huge from a design perspective, as making sure our designs look the same on different operating systems and browsers can be quite challenging, so anything that ensures uniformity regardless of the viewer's machine is a big win for us.

To learn how to use Google Fonts in a web page, check out the tutorial below:

Google Fonts Tutorial

 

LINEAR-GRADIENT

You may have also noticed that the background for the license is not just a single color; in fact, it seems to be made of multiple colors, smoothly transitioned from one to another. 

This is done by setting the background to use a special CSS function called "linear-gradient".  To learn more about how to use this, check out the documentation below!
