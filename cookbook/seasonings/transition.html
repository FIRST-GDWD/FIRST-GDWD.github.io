The transition property is a very fun CSS Property allows you to add simple animations between different states of an HTML element.

A Preface: HTML Element States

Before we dive into the transition property, we need to talk briefly about a key term from the introduction above.

What do we mean by "different states"? The simplest example would be something like a hover state.

Have you noticed that when you hover over a hyperlink the cursor changes to a pointer icon? That is because when the anchor enters the hover state, a CSS Property called "cursor" is changed from its default value to the "pointer" value.

How do we define or set up these different states? While custom states can be defined using JavaScript, the majority of basic HTML Element states can be targeted in CSS using something called pseudo-classes.

Pseudo-classes

Pseudo-classes are special keywords that can be added in CSS selectors to define a CSS Rule targeting an HTML Element in a certain state.

They have a bit of a funny syntax using the colon ( : ), but what's one more symbol added to the two selector symbols you already know (the hashtag # and dot .)?

Let's use our hover state as an example again. Suppose we wanted to change the hover state of ALL anchors on a page so that they turn turquoise when you hover over them. In that case, we might use some CSS code like this:

a:hover {
    color: turquoise;
}

That code targets ALL anchors on the page, and tells them to change to the color turquoise on hover.

Code like this is critical to using the transition CSS Property. Animation can only occur if there is a change of some sort on an HTML Element; otherwise, what would actually be animated? And pseudo-classes are one of the easiest ways to define different states for HTML Elements to animate through.

There are many different pseudo-classes you can use in your CSS code, some more useful than others. However, they are not the focus of this entry, so we'll just leave you with some extra additional resources at the bottom of the page if you're interested to learn more.

Alright, now let's get into the real meat and potatoes of this entry: the transition property.

The Anatomy of the Transition Property

The transition property is one of the more complex CSS Properties that you'll be learning about. It's not too bad once you get the hang of it, but until you do, it'll look kinda weird.

To use the transition property, you'll need to register other CSS properties with it, so that it knows what to animate.

To register a CSS Property with the transition property for animation, you'll need to include two things:

  1. The name of the CSS Property
  2. The duration (in seconds) of the animation

Let's take a look at a basic example, using the same objective we used above, changing anchor to turquoise when we hover over them:

a {
    transition: color 0.5s;
}

Inside there, you can see the two parts we said we needed above: the name of the property we want to animate (the color property) and duration we want the animation to have (0.5s, or in other words, a half second), each separated by a space.

CAVEAT: You can't use just any CSS Property with the transition property. You can only register CSS Properties that are considered "animatable". Most of time, if the property accepts color or number values, then it can be animated. But more complex properties, like the display property for example, are not "animatable", so they will not work if you try to register them with the transition property.

Simple Example: Animating One CSS Property

When we bring that code together with the code we wrote earlier, we will have all we need to animate anchors to turquoise on hover!

EXAMPLE

a {
    transition: color 0.5s;
}

a:hover {
    color: turquoise;
}

Try hovering over this and see!

The anchor above take a half second to animate smoothly between its normal color in its default state and the red color in its hover state. AND, it also takes a half second to animate back to its default state when you move your cursor off of the anchor.

Warning: Be Sure to Set the Transition Property on the Right State!

If you look closely at the example above, you might wonder why we didn't just include the transition property on the hover state. That seems like it'd be simpler.

Well, let's try it, and see what happens. We'll add an extra class and use a different color to separate this example from the previous one.

EXAMPLE

a.bad-example:hover {
    transition: color 0.5s;
    color: orange;
}

Hover on and off me!

When you hover over the hyperlink above, all seems well! It seems like nothing is wrong...until you move your cursor off the anchor.

Did you catch what happens? It's subtle, but it didn't animate back to its default state: it just instantly snapped back.

That's because it's doing exactly what we told it to do: we only put the transition property on the hover state, so it will ONLY transition (animate) when we hover over it. As soon as we move the cursor off of it, there is no transition property set, so it doesn't animate back.

This is an important part to using the transition property:

Set the transition property on the DEFAULT state of your HTML Element, NOT on the state that you're animating to! (in other words, don't set transition property on the pseudo-class selector!)

There are certain exceptions to this rule when you're doing complex animations, but most of the time, following this rule will keep your animations working properly!

Advanced Example: Animating Multiple CSS Properties

In the examples above, all we did was attempt to change the color of a hyperlink. But what if we wanted to animate more than one CSS Property?

Well, we can do that too! All we need to do is register more properties with the transition property.

Not that we'd want to actually do this in reality, but let's say that we want to animate the color, background-color, and font-size when we hover over the next paragraph.

EXAMPLE

p.advanced-example {
    transition:
        color 1s,
        background-color 1s,
        font-size 1s;
}

p.advanced-example:hover {
    color: white;
    background-color: navy;
    font-size: 24px;
}

It works! And this highlights a couple of important things:

  • Multiple CSS Properties can be registered with the transition property (just be sure to separate them with commas and not semicolons!)
  • Anchors are not the only things that can be animated or have hover states! Any HTML element can use the transition property, and any HTML element can use pseudo-classes like :hover
  • Animating the sizing or spacing of HTML elements can affect other elements around them.

That last one is important from both a design and a UI/UX perspective. Notice how when you hover over that advanced example paragraph, it pushes down everything beneath it? We usually want to avoid that.

Movement of content can be a bit distracting, but it doesn't usually affect too much if things grow vertically; however, if things also grow horizontally, it can often mess up your design, especially in smaller, more compact parts of your site, like a navbar.

So while it can be fun, just don't go too crazy with transitions when it comes to professional web design! Subtlety is usually a better approach.
